# Информация о .gitignore

## Что добавлено в .gitignore

### 📁 Исключенные папки:
- **`app_tests/`** - Инструменты и скрипты тестирования
- **`demo_data/`** - Демонстрационные данные
- **`demo_scenario/`** - Сценарии демонстрации

### 🔧 Другие исключения:
- Файлы результатов тестирования (`*_results_*.json`)
- Временные файлы тестов (`*_test_*.json`, `test_*.log`)
- Модели машинного обучения (`*.pkl`, `*.joblib`)
- Данные (`*.csv`, `*.json`, `*.xlsx`)
- Логи (`*.log`, `logs/`)
- Кеш файлы (`.cache/`, `*.cache`)
- Стандартные Python исключения (`__pycache__/`, `*.pyc`, etc.)

### 📋 Сохраненная структура:
Для сохранения структуры папок в git созданы `.gitkeep` файлы:
- `app_tests/.gitkeep`
- `demo_data/.gitkeep`
- `demo_scenario/.gitkeep`

## Зачем это нужно

1. **Чистый репозиторий** - исключает временные файлы и результаты тестов
2. **Безопасность** - не попадают в репозиторий данные и модели
3. **Производительность** - уменьшает размер репозитория
4. **Организация** - четкое разделение кода и вспомогательных материалов

## Что попадает в git

✅ **Включается:**
- Исходный код приложения
- Конфигурационные файлы
- Документация
- Docker файлы
- Схемы базы данных

❌ **Исключается:**
- Результаты тестирования
- Демонстрационные данные
- Временные файлы
- Логи выполнения
- Модели ML (большие файлы)

## Как работать с исключенными папками

Если нужно добавить конкретные файлы из исключенных папок:
```bash
git add -f app_tests/specific_file.py
```

Для просмотра игнорируемых файлов:
```bash
git status --ignored
```
