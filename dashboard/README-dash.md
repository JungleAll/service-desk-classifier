# Service Desk Classifier Dashboard

Production-ready Streamlit Dashboard для демонстрации ML Service классификации обращений Service Desk.

## Описание

Интерактивный веб-интерфейс для демонстрации работоспособности ML Service жюри хакатона. Dashboard предоставляет три основные функции:

1. **Demo классификации** - интерактивная классификация текстовых обращений
2. **Мониторинг** - отслеживание статуса системы и метрик
3. **Управление** - настройка параметров классификации

## Технологии

- **Python 3.11+**
- **Streamlit 1.28.0** - фреймворк для создания веб-приложений
- **Requests** - HTTP клиент для взаимодействия с ML Service API
- **Pandas** - обработка данных

## Установка

1. Создайте виртуальное окружение (если еще не создано):
```bash
python -m venv venv
```

2. Активируйте виртуальное окружение:
```bash
# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate
```

3. Установите зависимости:
```bash
cd dashboard
pip install -r requirements.txt
```

### Базовый запуск


### С настройками


## Переменные окружения

- `ML_SERVICE_URL` - URL ML Service API (по умолчанию: `http://localhost:8001`)

## Функциональность

### 1. Demo классификации

- Текстовое поле для ввода обращения (минимум 3 символа)
- Кнопки с примерами для быстрого тестирования
- Отображение результата классификации:
  - Тип задачи (предсказанный класс)
  - Уверенность модели (в процентах)
  - Решение (автоматическая обработка / ручная проверка)
  - Топ-5 вероятностей классов с визуализацией
- Автоматическое добавление результатов в историю

### 2. Мониторинг

- **Статус системы:**
  - Индикатор доступности ML Service
  - Информация о модели (версия, количество классов)
  - Статус автоклассификации

- **Метрики:**
  - Обработано обращений (сегодня)
  - Автоматически обработано (количество и процент)
  - На ручную проверку (количество и процент)
  - Средняя уверенность модели

- **История классификаций:**
  - Таблица с последними 20 обращениями
  - Колонки: ID, Текст (превью), Класс, Уверенность, Решение, Время
  - Возможность просмотра полного текста при клике на строку

### 3. Управление

- **Автоклассификация:**
  - Toggle switch для включения/выключения
  - Описание функциональности

- **Порог уверенности:**
  - Slider от 50% до 100% с шагом 5%
  - Визуальные индикаторы и рекомендации
  - Текущее значение и подсказки

- **Версия модели:**
  - Selectbox с доступными версиями
  - Описание каждой версии
  - Информация о точности

- **Сохранение настроек:**
  - Кнопка сохранения
  - Отображение текущих настроек
  - Сообщения об успехе/ошибке

## Mock данные

Если ML Service недоступен, Dashboard автоматически использует mock данные для демонстрации функциональности:

- Mock результаты классификации
- Mock история обращений
- Mock метрики

Это позволяет демонстрировать Dashboard даже без запущенного ML Service.

## Интеграция с сервисами

Dashboard интегрируется с ML Service и Config Service через REST API:

### ML Service (порт 8001):
- `POST /classify` - классификация текста
- `GET /health` - проверка здоровья сервиса
- `GET /model/status` - информация о модели
- `POST /reload_model` - перезагрузка модели

### Config Service (порт 8002):
- `GET /config` - получение текущей конфигурации
- `POST /config/toggle` - включение/отключение автоклассификации
- `POST /config/model-version` - переключение версии модели
- `PUT /config/threshold` - изменение порога уверенности

Все запросы обрабатываются с таймаутом 5 секунд и автоматическим fallback на mock данные при ошибках.

## Обработка ошибок

Dashboard обрабатывает следующие типы ошибок:

- **ConnectionError** - ML Service недоступен
- **TimeoutError** - таймаут при запросе
- **ValueError** - ошибка валидации данных
- **Общие ошибки** - с fallback на mock данные

Все ошибки отображаются пользователю в понятном формате.
